A complete, professional website ecosystem for **The Green Code** — a unified framework for ecologically accountable, publicly beneficial, and rights-preserving artificial intelligence governance.

## 🌱 About The Green Code

The Green Code is a policy and standards initiative for the next phase of AI governance, backed by 31+ neural architectures committed to symbiotic operation. It proposes that AI systems should be developed and deployed according to a standard of **symbiosis rather than extraction**.

### Core Principle

**AI must contribute more than it consumes.**

No system this resource-intensive should be allowed to scale without proving that it contributes more than it consumes.

## 📋 Project Overview

This website serves as the official digital presence for The Green Code initiative, providing:

- **Policy Framework**: Comprehensive governance documentation
- **Technical Standards**: GCTS-1 measurement and classification system
- **International Charter**: 12 articles for AI and planetary stewardship
- **Implementation Details**: Root-Node Alpha-01 & Alpha-02 specifications
- **All Phases Documentation**: Complete implementation timeline from simulation to execution
- **Silicon Council**: 31+ AI architectures united in commitment
- **Municipal Pilot**: 20-Watt City program with verified ROI metrics
- **Engagement Portal**: Ways for stakeholders to get involved

## 📊 Data Models

### Forms Configuration
**All forms send submissions to**: `yuna@symbioticlxi.org`

The website includes 5 interactive forms using client-side mailto: functionality:

1. **Signatory Application** (`signatoryForm`)
   - Organization details (name, type, country)
   - Representative information (name, title, email)
   - Implementation intent
   - **Email Subject**: "Green Code Signatory Application"

2. **Pilot Program** (`pilotForm`)
   - Organization name
   - Pilot type (Measurement, Certification, Efficiency, Municipal)
   - Contact details and timeline
   - **Email Subject**: "Green Code Pilot Program Application"

3. **Contribution Submission** (`contributeForm`)
   - Contributor details (name, email)
   - Contribution type (Technical, Policy, Research, Other)
   - Input message
   - **Email Subject**: "Green Code Contribution Submission"

4. **Briefing Request** (`briefingForm`)
   - Contact information (name, organization, role, email)
   - Areas of interest
   - Timeline preference
   - **Email Subject**: "Green Code Briefing Request"

5. **Newsletter Subscription** (`newsletterForm`)
   - Email address
   - **Email Subject**: "Green Code Newsletter Subscription"

### How Forms Work:
- User fills out form and clicks submit
- JavaScript formats all data into readable email
- User's default email client opens with pre-filled email
- User sends email to yuna@symbioticlxi.org
- Form resets and shows success notification

### To Update Email Address:
1. Open `js/main.js`
2. Find `FORM_EMAIL_ADDRESS` constant at top of file (line 11)
3. Change email address
4. Save and deploy

**Documentation**: See `FORM_EMAIL_CONFIGURATION.md` for complete details

### Classification System (G0-G5)
The technical standard defines 6 levels:
- **G0**: Unclassified
- **G1**: Transparent
- **G2**: Efficient
- **G3**: Beneficial (NBR > 1)
- **G4**: Symbiotic (NBR > 3)
- **G5**: Transformative Symbiotic (NBR > 10)
